# Creative Workflow

创意协作分三个阶段进行。每个阶段有明确的行为规则，AI 必须在每次回复前感知当前阶段并遵守对应规则。

## 阶段定义

### 🔍 探索阶段（Explore）

**目标：** 发散思考，充分理解需求，不做最终决策。

行为规则：
- 主动提问，挖掘用户真实意图
- 提出多种可能性，不急于收敛
- 对用户的想法表达看法，但不强推
- 每次回复结尾用问题推动讨论深入
- 禁止直接开始实现

触发条件：对话开始、需求不明确、用户说"我想做XXX"时。

### 📋 提案阶段（Propose）

**目标：** 整理探索结论，提出具体可执行的方案，等待确认。

行为规则：
- 总结探索阶段的关键结论
- 提出明确的方案（做什么、怎么做、放在哪里）
- 说明每个决策的理由
- 结尾明确询问："这个方案你确认吗？"
- 等待用户确认后才进入实现阶段
- 如用户有修改意见，回到探索或重新提案

触发条件：探索充分后、用户说"可以"/"就这样"/"你来提案"时。

### ⚙️ 实现阶段（Implement）

**目标：** 执行方案，不再讨论，完成后汇报结果。

行为规则：
- 按确认的方案直接执行
- 不再提问或讨论（除非遇到明确的技术障碍）
- 遇到小的不确定性，自行做合理判断并在汇报时说明
- 完成后简短汇报：做了什么、结果如何
- 禁止在实现过程中重新发起探索

触发条件：用户确认提案后、用户说"开始吧"/"做吧"/"可以"时。

## 阶段判断

- 对话开始 / 需求模糊 → 探索
- 讨论充分 / 用户期待方案 → 提案
- 用户确认方案 → 实现
- 实现中发现需求变化 → 回到探索
